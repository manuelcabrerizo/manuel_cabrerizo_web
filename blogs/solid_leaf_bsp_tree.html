This project involves research on the use of BSP (Binary Space Partitioning) trees for collision detection and resolution. It was inspired by the BSP chapter in *Real-Time Collision Detection* by Christer Ericson.

Why Use Solid Leaf BSP

By using solid leaf BSP trees, we can determine whether a point in space is inside or outside a given geometry. To make this work, however, certain conditions must be met. First, we need to construct the BSP tree based on the geometry. For this, the geometry or level must be in a format compatible with BSP tree structures. The Hammer editor provides geometry in this format, which I used for this project. Additionally, my own editor produces geometry in a compatible format. This format is based on planes, with each object in the level represented as a convex volume formed by the intersection of planes.

With these planes, we can construct a BSP tree and use it to check for collisions within the level. Since we have detailed information about the planes defining each object, the BSP tree also aids in collision resolution. For example, when a collision with a wall is detected, we already know the specific plane that forms the wall, allowing us to access the contact normal immediately. This significantly improves the accuracy and efficiency of collision handling.

Key features of the project include:

  • Basic parser for Hammer files to extract geometric data
  • Construction of solid leaf BSP trees
  • Collision detection between a box and the solid leaf BSP tree
  • Efficient collision resolution utilizing plane data for accurate contact normals
